## 1. 产品概述

### 1.1 产品背景
小红书创作者在日常中会产生大量碎片化灵感（如生活观察、金句、选题方向），但通常散落在备忘录、微信笔记或脑海中。等到需要创作时，又因缺乏相关参考素材或不知如何组织内容而难以落笔。市面上的 AI 写作工具多为通用型，缺乏对小红书平台调性（短平快、标题党、表情符号、个人体验分享）的深度适配，也无法利用用户自己的历史笔记和参考案例进行个性化创作。

### 1.2 产品定位
一款专为小红书创作者设计的轻量级 AI 创作辅助工具。提供 **灵感收集** + **素材库管理** + **双模式智能创作**（自动创作 / 协作创作），帮助用户将碎片化想法与高质量参考笔记相结合，快速生成符合平台风格的新笔记。

### 1.3 产品目标
- **核心目标**：将用户记录的灵感和导入的参考笔记转化为可直接发布的小红书笔记，减少从“想法”到“成稿”的时间。
- **体验目标**：记录灵感零摩擦，素材库易于检索，AI 生成内容保留个人风格且支持人工修改。
- **量化指标**：用户平均每周生成笔记 ≥3 篇；单次生成操作 ≤3 次点击；生成内容用户修改率 ≤30%（表示足够可用）。

|------|------|----------|

## 3. 核心功能需求

### 3.1 灵感收集 – 碎片化记录
- **快速记录入口**：提供 Web/移动端浮动按钮、快捷指令（如 iOS 小组件），点击后直接输入文本，无需打开完整应用。
- **支持多种形式**：文字（≤500 字）、图片（OCR 提取文字）、语音（转文字）。
- **自动打标签**：根据内容关键词智能推荐标签（如 #美妆 #穿搭 #探店），用户可修改或忽略。
- **灵感聚合视图**：按时间倒序排列，支持搜索、按标签筛选、置顶重要灵感。
- **灵感一键转创作**：每条灵感旁有“使用此灵感创作”按钮，直接跳转到创作模块。

### 3.2 素材库 – 参考笔记管理
- **导入方式**：
  - 手动粘贴小红书笔记链接 → 自动抓取正文、图片（可选）、标题、点赞数。
  - 直接粘贴文本。
  - 上传本地笔记截图（OCR 识别文字）。
  - 批量导入（CSV 文件包含标题+正文）。
- **素材库结构**：
  - 每条素材包含：标题、正文、来源（自己/他人）、标签、收藏时间。
  - 支持自定义字段（如“竞品分析”“爆款公式”）。
- **检索与筛选**：
  - 全文搜索（标题+正文）。
  - 按标签、来源、时间范围筛选。
  - 相似素材推荐（基于向量相似度）。
- **安全与隐私**：用户导入的“他人笔记”仅供个人参考，不对外分发；提供导入合规提示。

### 3.3 双模式智能创作

#### 3.3.1 自动创作模式
- **入口**：用户点击“自动创作”，可选择是否绑定某些灵感或素材。
- **流程**：
  1. 系统自动从用户的**灵感收集**中选取最近 3-5 条相关灵感（或用户手动勾选）。
  2. 从**素材库**中检索与这些灵感主题相似的参考笔记（最多 5 条）。
  3. AI 模型（如通义千问、GPT-4）根据以下输入生成完整笔记：
     - 用户灵感原文
     - 参考笔记的风格特点（标题写法、表情符号使用频率、段落长度）
     - 小红书通用写作规则（带话题标签、开头吸引力、正文有个人体验感、结尾引导互动）
  4. 生成结果包括：**标题 + 正文 + 推荐话题标签** + 封面建议（文字描述）。
- **用户控制**：可指定笔记类型（种草/攻略/日常/Vlog文案）、目标字数（200-800字）、语气（活泼/专业/真诚）。

#### 3.3.2 协作模式
- **入口**：用户选择“协作创作”，需主动提供：
  - 本次想要表达的核心观点 / 需求（例如：“想推荐一款平价粉底液，适合油皮”）
  - 从素材库中手动添加 1-10 条参考笔记（或勾选全部相关素材）
- **交互流程**：
  1. AI 先分析用户提供的参考笔记，提取共同风格要素和关键卖点。
  2. 生成**草稿大纲**（如：痛点引入→产品介绍→使用对比→总结推荐），用户可修改大纲。
  3. 用户与 AI 以对话形式协作：用户可以对某一段落提出修改要求（“换一个更夸张的开头”“增加表情符号”）。
  4. 最终生成符合用户要求的小红书笔记，且保留个人语气。
- **版本对比**：支持查看生成过程中不同版本的差异，用户可以回退。

### 3.4 笔记管理与导出
- 所有生成的笔记自动保存到“我的笔记库”，支持再次编辑、重新生成、收藏。
- 导出格式：纯文本、富文本（保留加粗/颜色）、小红书 App 直接分享（通过剪贴板或 API）。
- 一键复制标题+正文+标签，适配小红书编辑框。

## 4. 非功能需求

### 4.1 性能要求
- 灵感记录响应时间 ≤ 0.5 秒。
- 自动创作模式从点击到生成完整笔记的时间 ≤ 10 秒（含 AI 推理）。
- 素材库链接抓取成功率 ≥ 95%（针对有效小红书公开链接）。

### 4.2 可用性与体验
- 移动端适配良好（响应式 Web 或原生 App）。
- 灵感记录支持离线缓存，联网后自动同步。
- 创作过程中允许中断，自动保存草稿。

### 4.3 数据安全与隐私
- 用户的所有灵感、素材库、生成笔记仅存储在用户本地或私有云，不允许平台抓取用于训练其他用户模型。
- 若使用第三方 AI API，传输过程加密，且承诺不将用户数据用于模型训练（使用 Azure OpenAI 数据隔离承诺等）。
- 用户可一键导出所有数据并完全删除账户。

### 4.4 可扩展性
- 后期可集成小红书发布 API（如有权限），实现“生成+发布”闭环。
- 支持接入更多大语言模型（切换模型无需重写业务逻辑）。

## 5. 用户交互流程示例

### 场景一：灵感记录 + 自动创作
1. 用户逛超市时突然有一个灵感：“油皮夏天找粉底液好难，需要遮瑕又不闷痘”。
2. 打开 App 灵感收集模块，用语音输入这句话，系统自动加标签 #油皮 #粉底液。
3. 一周后，用户准备写一篇粉底液推荐笔记，打开“自动创作”。
4. 勾选灵感“油皮夏天...”，系统自动匹配素材库中 3 篇高赞粉底液笔记。
5. 点击“生成”，10 秒后得到一篇标题为“油皮亲妈！夏天24h不脱妆的粉底液实测”，正文包含 5 款产品对比 + 真实上脸感受 + 注意事项。
6. 用户稍作修改，一键复制到小红书发布。

### 场景二：协作模式
1. 用户想写一篇“露营装备清单”，自己手头有两个参考笔记，另从他人笔记中导入三篇爆款。
2. 进入协作模式，输入需求：“适合新手、预算 2000 元以内、风格要轻松幽默”。
3. AI 生成大纲（帐篷→睡袋→炉具→小工具）。
4. 用户对“炉具”部分不满意，要求“加上一句防踩坑：不要买那种一体式炉头，容易倒”。
5. AI 即时修改，最终生成完整笔记。
6. 用户保存并导出。

## 6. 技术架构建议（简述）

- **前端**：Flutter（跨平台移动端）或 React + Taro（小程序/H5）。
- **后端**：Node.js / Python FastAPI，提供 REST API 与 WebSocket（协作模式流式生成）。
- **AI 层**：接入阿里通义千问或 GPT-4，使用 LangChain 实现 RAG（素材库检索增强）和对话式生成。
- **素材库检索**：使用向量数据库（如 Chroma、Pinecone）存储笔记片段的 Embedding，支持语义相似度匹配。
- **数据存储**：用户数据使用云原生数据库（MongoDB 或 PostgreSQL）+ 对象存储（图片）。
- **链接抓取**：服务端模拟请求解析小红书公开页面（需注意反爬策略，建议使用官方开放接口如果存在）。

## 8. 成功指标（KPI）

- **周活跃用户平均生成笔记数** ≥ 3 篇。
- **自动创作模式使用率** ≥ 60% 的活跃用户。
- **协作模式留存率**：用过协作模式的用户次周留存率比未用者高 20%。
- **创作耗时**：从打开应用到生成可发布笔记平均 ≤ 5 分钟。
- **用户满意度**：内测 NPS ≥ 40。
